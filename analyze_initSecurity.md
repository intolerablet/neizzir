# Анализ логики функции initSecurity 

## Общая структура функции initSecurity (0x000d7c10)

### 1. Инициализация параметров
```c
FUN_000d86e0(abStack_64, &puStack_58);        // param1
FUN_000d86e0(abStack_70, &puStack_58, param_4); // param2
FUN_000d86e0(abStack_7c, &puStack_58, param_5); // param3
```

### 2. ПЕРВЫЙ HTTP запрос (к первому IP)
```c
uVar2 = FUN_000946d0(DAT_001dd0a0, abStack_70, abStack_64, abStack_7c);
```
- `DAT_001dd0a0` - указатель на первый IP (194.135.36.88)
- Возвращает результат в `uVar2`

### 3. Если первый запрос FAILED (uVar2 & 1 == 0):

#### 3.1. Показывает сообщение об ошибке
```c
FUN_000e9bf0(&DAT_001dd2d0);  //расшифровывает и показывает ошибку
```

#### 3.2. Освобождает память первого IP
```c
if (DAT_001dd0a0 != NULL) {
    operator.delete(pbVar1);
}
```

#### 3.3. Переключается на ВТОРОЙ IP (резервный)
```c
DAT_001dd0a0 = FUN_000d87a0(&DAT_001dd34c);  // второй IP: 162.19.254.18 
uVar2 = FUN_000946d0(DAT_001dd0a0, ...);     // повторный запрос
```

### 4. Обработка результата запроса
```c
FUN_0009602c(uVar2 & 1, (uVar2 << 0x10) >> 0x18, (uVar2 << 8) >> 0x18);
```

### 5. Анализ кода ответа (uVar2):

#### Бит 0 (uVar2 & 1):
- `0` = запрос failed
- `1` = запрос success

#### Биты 8-15 ((uVar2 >> 8) & 0xff):
- `0` = специальная обработка (показывает разные ошибки)
- `1-2` = показывает другое сообщение
- Другие значения = показывает третье сообщение

### 6. Если запрос FAILED снова (uVar2 & 1 == 0):

Показывает одно из сообщений об ошибке в зависимости от кода:

**Код 0 (uVar2 >> 8 & 0xff == 0):**
- Если бит 16 установлен:
  - Если бит 17 установлен: показывает ошибку из `DAT_001dd3d0`
  - Иначе: показывает ошибку из `DAT_001dd410`
- Иначе: показывает ошибку из `DAT_001dd3a0`

**Код 1-2:**
- Показывает ошибку из `DAT_001dd440`

**Другие коды:**
- Показывает ошибку из `DAT_001dd480`

### 7. Если запрос SUCCESS (uVar2 & 1 == 1):

Создает новый поток:
```c
pthread_create(&uStack_50, NULL, 0xd6b41, NULL);
```
- Адрес функции потока: `0x000d6b40` (LAB_000d6b40+1)

## ВОЗМОЖНЫЕ ЗАЩИТЫ

### Нет проверки целостности библиотеки
- Нет CRC/MD5/SHA проверок
- Нет anti-tamper защиты

### Нет проверки оригинального IP
- IP просто расшифровывается и используется
- Нет hardcoded проверок типа "if (ip != 194.135.36.88) exit()"

### Функция HTTP запроса (FUN_000946d0)
Может содержать:
- Проверку SSL сертификата
- Проверку подписи ответа сервера
- Валидацию JSON структуры

**НУЖНО ПРОВЕРИТЬ:** Что делает `FUN_000946d0`

### Функция обработки результата (FUN_0009602c)
```c
FUN_0009602c(success, code_high, code_mid);
```
Может сохранять результат в глобальную переменную или SharedPreferences
